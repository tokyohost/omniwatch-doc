# 第 12 课：用中文点阵显示天气屏幕

[上一课](11-天气屏幕进阶教程.md) · [返回总目录](../README.md) · [性能优化篇](13-性能优化篇.md)

上一课为了保证兼容性，把城市和天气描述写成了英文。这一课继续改造天气屏幕：用点阵字模显示中文，把 `Shanghai`、`Cloudy`、`HUM`、`WIND` 等内容替换成 `上海`、`多云`、`湿度`、`风速`。

核心思路很简单：不要把完整中文字体塞进设备，而是只把本屏幕会用到的汉字提前转换成点阵数据，再用 `canvas.pixel()` 按点绘制。

本课完整示例已经放在：

```text
example/code/chinese_weather_screen/
├── custom_data_plugin/
│   └── weather_cn/
│       ├── plugin.json
│       ├── main.py
│       └── requirements.txt
└── styles/
    └── style_weather_cn.py
```

其中 `custom_data_plugin/weather_cn` 是输出中文城市和中文天气描述的自定义数据插件，`styles/style_weather_cn.py` 是中文点阵天气屏幕样式。样式文件已经内置天气屏幕常用汉字的 `FONT_16` 点阵字模，Pico 不需要自带中文字库。

## 为什么要用点阵字模

内置 `canvas.text()` 适合英文、数字和少量内置字符。中文字符数量很大，完整字体文件通常远超自定义样式大小限制。点阵字模的做法是：

1. 在电脑上用字体文件把指定汉字渲染成黑白点阵。
2. 把点阵转换成 Python 字典。
3. 把字典粘贴进自定义样式。
4. 样式运行时逐点绘制这些汉字。

这种方式适合标题、城市、天气状态、标签等少量中文。不要把一整篇文章或完整中文字库放进样式文件。

## 第一步：准备中文转点阵脚本

本教程提供了脚本：

```text
example/code/chinese_text_to_bitmap.py
```

脚本依赖 Pillow。开发电脑上执行：

```bash
python -m pip install pillow
```

Windows 常用中文字体路径示例：

```text
C:/Windows/Fonts/msyh.ttc
C:/Windows/Fonts/simsun.ttc
C:/Windows/Fonts/simhei.ttf
```

如果字体路径不存在，打开 `C:\Windows\Fonts` 查看实际字体文件名。

## 第二步：准备需要显示的中文

只生成本屏幕会用到的字符。例如本课要显示：

```text
天气 上海 北京 深圳 广州 杭州 晴 多云 阴 小雨 中雨 大雨 雷雨 雪 雾 湿度 风速 更新 等待数据 插件异常
```

其中有些词不会同时出现，但提前放进去方便后面切换城市和天气状态。字符越多，样式文件越大；最终上传前可以删掉不用的字。

## 第三步：生成 FONT_16 字模

在 `omniwatch-doc` 目录中运行：

```bash
python example/code/chinese_text_to_bitmap.py "天气上海北京深圳广州杭州晴多云阴小雨中雨大雨雷雨雪雾湿度风速更新等待数据插件异常" --font "C:/Windows/Fonts/msyh.ttc" --size 16 --box-size 16 --format python-hex --name FONT_16 --output font_16.py --preview
```

如果你在 Linux 中运行，可以改成系统字体路径，例如：

```bash
python example/code/chinese_text_to_bitmap.py "天气上海多云湿度风速更新等待数据插件异常" --font "/usr/share/fonts/truetype/noto/NotoSansCJK-Regular.ttc" --size 16 --box-size 16 --format python-hex --name FONT_16 --output font_16.py --preview
```

输出文件 `font_16.py` 会包含类似结构：

```python
FONT_16 = {
    "天": ("0000", "0000", "0ff0", "..."),
}
```

打开 `font_16.py`，把缺失字符对应的条目复制到样式文件的 `FONT_16` 中。不要导入外部文件，因为上传自定义屏幕时通常只上传单个 `style_*.py` 文件。

## 第五步：编写中文绘制函数

新建 `style_weather_cn.py`。示例文件 `example/code/chinese_weather_screen/styles/style_weather_cn.py` 已经内置常用天气汉字的 `FONT_16`，可以直接上传测试。如果你换成其它城市或天气文案，再用上一步脚本给缺失汉字追加点阵。

```python
"""提供使用中文点阵字模显示天气数据的 OmniWatch 屏幕样式。"""

from config import BLACK, BLUE, DARK, GRAY, GREEN, HEIGHT, WHITE, WIDTH, YELLOW
from styles.style_plugins import register_style


FONT_16 = {
    "天": ("0000", "0000", "0ff0", "..."),
    "气": ("0000", "0000", "1f80", "..."),
}


class WeatherCnStyle:
    """绘制中文城市、天气描述、湿度、风速和更新时间。"""

    name = "weather_cn"
    zh_name = "中文天气"
    type = "custom"
    width = WIDTH
    height = HEIGHT
    landscape = False
    font_name = "native"

    @staticmethod
    def create_dirty_regions():
        """声明中文天气主体和页脚两个动态刷新区域。"""
        return [
            ("weather", 8, 52, 224, 180),
            ("footer", 8, 282, 224, 28),
        ]

    @classmethod
    def select_dirty_regions(cls, previous, current):
        """根据天气数据和系统时间变化选择需要重绘的区域。"""
        previous = previous or {}
        current = current or {}
        regions = {item[0]: item for item in cls.create_dirty_regions()}
        selected = []
        previous_weather = previous.get("ext", {}).get("weather", {})
        current_weather = current.get("ext", {}).get("weather", {})
        if previous_weather != current_weather:
            selected.append(regions["weather"])
        if previous.get("timestamp") != current.get("timestamp"):
            selected.append(regions["footer"])
        return selected

    @staticmethod
    def _number(value, default=0):
        """把未知输入安全转换为浮点数。"""
        try:
            return float(value)
        except (TypeError, ValueError):
            return float(default)

    @staticmethod
    def _read_weather(snapshot):
        """从 snapshot.ext.weather 中读取天气对象。"""
        return (snapshot or {}).get("ext", {}).get("weather", {}) or {}

    @staticmethod
    def _draw_bitmap_char(canvas, x, y, char, color, scale=1):
        """按点阵数据绘制单个中文字符，并返回字符占用宽度。"""
        rows = FONT_16.get(char)
        if not rows:
            return 0
        width = 16
        for row_index, row_hex in enumerate(rows):
            row_bits = int(row_hex, 16)
            for col_index in range(width):
                if row_bits & (1 << (width - 1 - col_index)):
                    if scale == 1:
                        canvas.pixel(x + col_index, y + row_index, color)
                    else:
                        canvas.fill_rect(
                            x + col_index * scale,
                            y + row_index * scale,
                            scale,
                            scale,
                            color,
                        )
        return width * scale

    @classmethod
    def _draw_bitmap_text(cls, canvas, x, y, text, color, scale=1, spacing=1):
        """按点阵字模绘制中文文本，缺失字会被跳过。"""
        cursor_x = x
        for char in str(text or ""):
            width = cls._draw_bitmap_char(canvas, cursor_x, y, char, color, scale)
            if width > 0:
                cursor_x += width + spacing
            elif char == " ":
                cursor_x += 4 * scale
        return cursor_x - x

    def _draw_header(self, canvas):
        """绘制中文标题和分隔线。"""
        self._draw_bitmap_text(canvas, 8, 12, "天气", BLUE, 1, 2)
        canvas.line(8, 42, 232, 42, DARK)

    def _draw_weather(self, canvas, snapshot):
        """绘制中文天气主体内容。"""
        weather = self._read_weather(snapshot)
        if weather.get("status") != "ok":
            self._draw_bitmap_text(canvas, 8, 72, "等待数据", YELLOW)
            self._draw_bitmap_text(canvas, 8, 96, "插件异常", GRAY)
            return

        city = str(weather.get("city", "--"))
        text = str(weather.get("text", "--"))
        temperature = self._number(weather.get("temperature_c"))
        humidity = self._number(weather.get("humidity"))
        wind = self._number(weather.get("wind_kmh"))
        updated_at = str(weather.get("updated_at", "--:--"))

        self._draw_bitmap_text(canvas, 8, 60, city, WHITE)
        canvas.text(8, 92, "{:4.1f} C".format(temperature), GREEN, 3)
        self._draw_bitmap_text(canvas, 8, 140, text, WHITE, 1, 2)
        canvas.fill_rect(8, 176, 224, 2, DARK)
        self._draw_bitmap_text(canvas, 8, 194, "湿度", GRAY)
        canvas.text(48, 194, "{:3.0f}%".format(humidity), GRAY)
        self._draw_bitmap_text(canvas, 112, 194, "风速", GRAY)
        canvas.text(152, 194, "{:4.1f}".format(wind), GRAY)
        self._draw_bitmap_text(canvas, 8, 220, "更新", BLUE)
        canvas.text(48, 220, updated_at, BLUE)

    @staticmethod
    def _draw_footer(canvas, snapshot):
        """绘制系统当前时间，方便确认屏幕仍在刷新。"""
        timestamp = str((snapshot or {}).get("timestamp", ""))
        clock = timestamp[11:19] if len(timestamp) >= 19 else "--:--:--"
        canvas.line(8, 280, 232, 280, DARK)
        canvas.text(8, 292, clock, GRAY)

    def draw_visible(self, canvas, snapshot):
        """绘制当前条带中可见的完整中文天气屏幕。"""
        snapshot = snapshot or {}
        canvas.clear(BLACK)
        self._draw_header(canvas)
        self._draw_weather(canvas, snapshot)
        self._draw_footer(canvas, snapshot)

    def draw_dirty(self, canvas, key, snapshot):
        """按区域键重绘天气主体或页脚。"""
        snapshot = snapshot or {}
        canvas.clear(BLACK)
        if key == "weather":
            self._draw_weather(canvas, snapshot)
        elif key == "footer":
            self._draw_footer(canvas, snapshot)


def create_weather_cn_style():
    """创建中文天气样式实例。"""
    return WeatherCnStyle()


register_style(WeatherCnStyle.name, create_weather_cn_style)
```

注意：Pico 没有中文字库时，所有中文都必须来自 `FONT_16` 点阵。示例样式已经内置常用汉字；新增城市或文案时，必须把新增汉字也追加进 `FONT_16`。

## 第六步：上传前控制文件大小

自定义样式文件大小有上限，点阵字模会明显增加体积。建议按下面顺序压缩：

1. 只生成实际会显示的汉字。
2. 天气描述尽量短，例如用 `小雨`，不要用 `小到中雨`。
3. 字号先用 `16`，不要一开始就用 `24` 或更大。
4. `scale=2` 会把像素放大，但字模本身仍然只有一份，适合标题少量放大。
5. 上传前检查文件大小，超过限制就减少字符或字号。

如果只显示 `天气、上海、多云、湿度、风速、更新、等待数据、插件异常` 这些字，通常比完整中文字库小很多。

## 第七步：上传和验证

1. 保存 `style_weather_cn.py`，确认 UTF-8 无 BOM。
2. 上传到“自定义屏幕”。
3. 在设备上选择“中文天气”。
4. 先看标题 `天气` 是否显示。
5. 再测试插件，确认 `snapshot.ext.weather.city` 是 `上海`，`text` 是 `多云` 这类中文。
6. 如果某个字空白，重新运行脚本，把那个字加入生成文本。

## 常见问题

| 现象 | 原因 | 处理方法 |
| --- | --- | --- |
| 中文完全不显示 | `FONT_16` 为空，或上传的不是内置字模版本 | 使用 `example/code/chinese_weather_screen/styles/style_weather_cn.py`，确认文件内有 `FONT_16` 条目 |
| 某个字缺失 | 生成字模时没有包含这个字 | 重新运行脚本，把缺失字加入文本 |
| 字太小 | 字号或缩放太小 | 生成时使用 `--size 18`，或绘制时把 `scale` 改为 `2` |
| 文件太大无法上传 | 字符太多或字号太大 | 减少字符、缩短天气文案、降低字号 |
| 中文边缘发虚 | 字体抗锯齿后阈值不合适 | 调整 `--threshold`，例如改成 `128` 或 `160` |
| 中文位置错乱 | 忘记给脏区域留够高度 | 扩大 `weather` 区域，重绘前保持 `canvas.clear(BLACK)` |

点阵中文的关键不是“支持所有汉字”，而是“只支持当前界面需要的汉字”。这样屏幕稳定、文件可上传，也更容易排错。

# 如何查看和导出日志

[返回总目录](../README.md) · [上传与排错](06-上传与排错.md) · [Dev 模式说明](09-Dev模式.md)

日志是程序按时间写下的运行记录。它能回答三个关键问题：电脑端是否找到设备、发送了什么、设备返回了什么。遇到黑屏、上传失败、样式回退或连接中断时，先看日志，通常比反复重启更快。

## Windows：直接从托盘查看

1. 在任务栏右下角找到 OmniWatch 托盘图标。
2. 如果图标被折叠，先单击 `^` 展开隐藏图标。
3. 右键单击 OmniWatch 图标。
4. 选择“打开日志”。
5. “系统监控日志”窗口会每秒自动刷新。

日志窗口默认显示最近约 1 MB 内容，并提供两个按钮：

- “复制全部”：将窗口中的日志复制到剪贴板，适合粘贴少量内容。
- “导出日志”：导出最近约 1 MB 日志，并在资源管理器中选中文件。

> 日志正在不断增加时，先复现一次问题，再马上打开或导出日志，最容易找到相关记录。

## Windows：直接找到日志文件

日志文件固定在当前 Windows 用户的本地应用数据目录：

```text
%LOCALAPPDATA%\PicoMonitor\pico-monitor.log
```

打开方法：

1. 按 `Win + R`。
2. 输入 `%LOCALAPPDATA%\PicoMonitor`。
3. 按回车。
4. 用文本编辑器打开 `pico-monitor.log`。

通过托盘“日志导出”生成的文件位于：

```text
%LOCALAPPDATA%\PicoMonitor\exports\
```

导出文件名带有时间，例如 `pico-monitor-20260707-163045-123456.log`。

## 自定义样式上传日志

上传样式时还有一块更聚焦的日志：

1. 右键托盘图标，打开“自定义屏幕”。
2. 单击“上传文件”并选择样式。
3. 查看窗口中的“上传日志”。

这里会显示文件校验、上传进度和设备响应。它适合判断本次上传为什么失败；“系统监控日志”则包含连接、采集和设备通信的完整上下文。上传问题最好两处都看。

## Linux：查看 systemd 服务日志

通过安装包运行时，OmniWatch Monitor 通常由 `pico-monitor` systemd 服务管理。

实时跟踪新日志：

```bash
sudo journalctl -u pico-monitor -f
```

查看最近 100 行并立即返回：

```bash
sudo journalctl -u pico-monitor -n 100 --no-pager
```

查看本次开机以来的日志：

```bash
sudo journalctl -u pico-monitor -b --no-pager
```

按 `Ctrl + C` 可退出实时跟踪，这不会停止后台服务。

## 从源码运行时怎么看

在终端运行 `pico_monitor.py` 时，日志直接打印在当前终端：

```powershell
python pico_monitor.py
```

不要关闭这个窗口。先运行程序，再复现问题，观察最新输出。需要保存时，可使用终端自身的复制功能；密码、用户名、IP、公网地址等隐私信息应在分享前打码。

## 如何读懂一行日志

常见格式如下：

```text
2026-07-07 16:30:45,123 [INFO] Pico LCD 已连接：COM3
```

从左到右分别是时间、级别和消息：

| 级别 | 含义 | 建议 |
| --- | --- | --- |
| `INFO` | 正常过程记录 | 用于确认程序走到了哪一步 |
| `WARNING` | 出现异常但程序可能继续 | 重点查看其后是否恢复 |
| `ERROR` | 操作失败 | 连同前后几十行一起分析 |

通信方向标记尤其有用：

- `[Monitor -> Pico]`：电脑端发给 OmniWatch。
- `[Pico -> Monitor]`：OmniWatch 返回给电脑端。
- `[DEV][Monitor -> Pico][JSON]`：Dev 模式中准备发送的数据快照。

例如只有 `[Monitor -> Pico]`、长期没有 `[Pico -> Monitor]`，通常说明设备未响应、串口不正确或通信中断。

## 按问题搜索关键词

| 问题 | 建议搜索 |
| --- | --- |
| 找不到设备 | `正在搜索`、`握手`、`PONG`、`通信异常` |
| 样式上传失败 | `uploadStyle`、`CUSTOM_STYLE`、`上传失败` |
| 自定义样式回退 | `自定义样式`、`回退`、`画布容量` |
| 数据是否发送成功 | `[Monitor -> Pico]`、`ACK:JSON` |
| Dev 模式状态 | `开发模式=开启`、`开发模式=关闭`、`[DEV]` |
| 程序启动配置 | `监控服务启动` |

## 提交问题时怎样提供日志

1. 记录问题发生的大致时间。
2. 导出日志，而不是拍一张模糊的屏幕照片。
3. 保留问题发生前后至少 30 行。
4. 同时说明 Windows/Linux 版本、设备是否连接、使用的样式名和复现步骤。
5. 分享前删除密码、令牌、用户名、公网 IP 等隐私信息。

不要只截取最后一行错误。真正的原因经常藏在它前面的连接或校验记录里。

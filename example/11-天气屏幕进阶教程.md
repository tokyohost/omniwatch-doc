# 第 11 课：进阶实战：自动获取天气并显示到屏幕

[上一课](10-自定义数据.md) · [下一课](12-中文点阵天气屏幕.md) · [返回总目录](../README.md) · [接口速查](07-Canvas接口速查.md)

这一课把前面的两条能力连起来：

1. 使用自定义数据插件在电脑端定时请求天气接口。
2. 把天气结果写入 `snapshot.ext.weather`。
3. 使用自定义屏幕样式读取 `snapshot.ext.weather` 并绘制温度、天气、风速、更新时间和数据状态。

示例使用 Open-Meteo 的天气预报接口。它支持用经纬度查询天气，教程中的请求不需要 API Key。正式使用前建议阅读接口文档，确认字段名称、访问频率和服务条款：https://open-meteo.com/en/docs

本课完整示例已经放在：

```text
example/code/weather_screen/
├── custom_data_plugin/
│   └── weather/
│       ├── plugin.json
│       ├── main.py
│       └── requirements.txt
└── styles/
    └── style_weather_panel.py
```

其中 `custom_data_plugin/weather` 是电脑端自定义数据插件目录，`styles/style_weather_panel.py` 是上传到“自定义屏幕”的样式文件。

## 最终效果

完成后，屏幕会显示类似下面的信息：

```text
WEATHER
Shanghai

27.8 C
Cloudy

Wind 12.4 km/h
Updated 14:35
```

电脑端负责联网和解析接口，设备端只负责显示已经整理好的小 JSON。这样做有三个好处：

- Pico 不需要直接访问互联网。
- 天气接口失败时不会拖慢屏幕绘制。
- 屏幕样式只读稳定字段，代码更短也更安全。

## 数据流

```text
Open-Meteo API
      ↓
电脑端天气插件 weather.collect()
      ↓
snapshot.ext.weather
      ↓
自定义屏幕 style_weather_panel.py
      ↓
OmniWatch 屏幕
```

## 第一步：创建天气插件目录

在电脑上新建一个目录，名称建议为 `weather`。也可以直接复制本课提供的示例目录：

```text
weather/
├── plugin.json
├── main.py
└── requirements.txt
```

开发时可以直接加载这个目录；分发给别人时再压缩成 ZIP。

## 第二步：编写 plugin.json

在 `weather/plugin.json` 中写入。示例文件位于 `example/code/weather_screen/custom_data_plugin/weather/plugin.json`：

```json
{
  "protocol": 1,
  "key": "weather",
  "name": "weather",
  "zh_name": "天气信息",
  "interval": 600,
  "entry": "main.py"
}
```

字段要点：

| 字段 | 本例取值 | 说明 |
| --- | --- | --- |
| `key` | `weather` | 数据最终写入 `snapshot.ext.weather` |
| `name` | `weather` | 采集任务名是 `custom_data.weather` |
| `zh_name` | `天气信息` | Windows 管理窗口里显示的名称 |
| `interval` | `600` | 默认每 600 秒采集一次，天气数据不需要每秒刷新 |

如果你已经有另一个插件使用了 `weather` 这个 `key` 或 `name`，必须改成唯一名称，例如 `home_weather`。

## 第三步：编写 requirements.txt

在 `weather/requirements.txt` 中写入。示例文件位于 `example/code/weather_screen/custom_data_plugin/weather/requirements.txt`：

```text
requests==2.32.3
```

Monitor 会为这个插件创建独立虚拟环境，所以它不会影响其他插件或电脑端程序本身。

## 第四步：编写 main.py

在 `weather/main.py` 中写入下面的完整代码。保存时选择 **UTF-8（无 BOM）**。示例文件位于 `example/code/weather_screen/custom_data_plugin/weather/main.py`。

```python
#!/usr/bin/env python3
"""采集 Open-Meteo 天气数据并转换为 OmniWatch 易显示的字段。"""

import datetime as dt

import requests


LATITUDE = 31.2304
LONGITUDE = 121.4737
CITY_NAME = "Shanghai"
TIMEZONE = "Asia/Shanghai"
API_URL = "https://api.open-meteo.com/v1/forecast"
USE_SYSTEM_PROXY = False

WEATHER_TEXT = {
    0: "Clear",
    1: "Mainly clear",
    2: "Partly cloudy",
    3: "Cloudy",
    45: "Fog",
    48: "Rime fog",
    51: "Light drizzle",
    53: "Drizzle",
    55: "Heavy drizzle",
    61: "Light rain",
    63: "Rain",
    65: "Heavy rain",
    71: "Light snow",
    73: "Snow",
    75: "Heavy snow",
    80: "Rain shower",
    81: "Shower",
    82: "Heavy shower",
    95: "Thunderstorm",
}


def _weather_text(code):
    """把天气代码转换为适合点阵屏显示的英文短文本。"""
    return WEATHER_TEXT.get(code, "Unknown")


def _hour_minute(value):
    """从接口时间字符串中提取小时和分钟。"""
    try:
        parsed = dt.datetime.fromisoformat(str(value))
        return parsed.strftime("%H:%M")
    except ValueError:
        return "--:--"


def collect():
    """请求天气接口并返回可写入 snapshot.ext.weather 的 JSON 对象。"""
    params = {
        "latitude": LATITUDE,
        "longitude": LONGITUDE,
        "timezone": TIMEZONE,
        "current": "temperature_2m,relative_humidity_2m,weather_code,wind_speed_10m",
    }
    try:
        session = requests.Session()
        session.trust_env = USE_SYSTEM_PROXY
        response = session.get(API_URL, params=params, timeout=(3, 6))
        response.raise_for_status()
        payload = response.json()
        current = payload.get("current") or {}
        weather_code = current.get("weather_code")

        return {
            "city": CITY_NAME,
            "temperature_c": current.get("temperature_2m"),
            "humidity": current.get("relative_humidity_2m"),
            "wind_kmh": current.get("wind_speed_10m"),
            "code": weather_code,
            "text": _weather_text(weather_code),
            "updated_at": _hour_minute(current.get("time")),
            "status": "ok",
        }
    except requests.RequestException as exc:
        return {
            "city": CITY_NAME,
            "text": "Network error",
            "updated_at": dt.datetime.now().strftime("%H:%M"),
            "status": "error",
            "message": str(exc)[:80],
        }
```

需要换城市时，修改这四行：

```python
LATITUDE = 31.2304
LONGITUDE = 121.4737
CITY_NAME = "Shanghai"
TIMEZONE = "Asia/Shanghai"
```

经纬度可以从地图工具中获取。`CITY_NAME` 建议使用英文或数字，因为当前点阵字体对中文覆盖有限，中文可能显示成 `?`。

## 第六步：编写天气屏幕样式

新建样式文件 `style_weather_panel.py`。文件名必须和 `name = "weather_panel"` 对应。示例文件位于 `example/code/weather_screen/styles/style_weather_panel.py`。

```python
"""提供显示自定义天气数据的 OmniWatch 屏幕样式。"""

from config import BLACK, BLUE, DARK, GRAY, GREEN, HEIGHT, WHITE, WIDTH, YELLOW
from styles.style_plugins import register_style


class WeatherPanelStyle:
    """绘制城市、温度、天气描述、湿度、风速和更新时间。"""

    name = "weather_panel"
    zh_name = "天气面板"
    type = "custom"
    width = WIDTH
    height = HEIGHT
    landscape = False
    font_name = "native"

    @staticmethod
    def create_dirty_regions():
        """声明天气内容和页脚时钟两个动态刷新区域。"""
        return [
            ("weather", 8, 52, 224, 180),
            ("footer", 8, 282, 224, 28),
        ]

    @classmethod
    def select_dirty_regions(cls, previous, current):
        """当天气数据或系统时间变化时选择需要重绘的区域。"""
        previous = previous or {}
        current = current or {}
        regions = {item[0]: item for item in cls.create_dirty_regions()}
        selected = []
        previous_weather = previous.get("ext", {}).get("weather", {})
        current_weather = current.get("ext", {}).get("weather", {})
        if previous_weather != current_weather:
            selected.append(regions["weather"])
        if previous.get("timestamp") != current.get("timestamp"):
            selected.append(regions["footer"])
        return selected

    @staticmethod
    def _number(value, default=0):
        """把未知输入安全转换为浮点数。"""
        try:
            return float(value)
        except (TypeError, ValueError):
            return float(default)

    @staticmethod
    def _short_text(value, limit):
        """把任意值转换为适合小屏显示的短文本。"""
        text = str(value or "--")
        return text[:limit]

    @staticmethod
    def _read_weather(snapshot):
        """从 snapshot.ext.weather 中读取天气对象。"""
        return (snapshot or {}).get("ext", {}).get("weather", {}) or {}

    def _draw_header(self, canvas):
        """绘制固定标题和分隔线。"""
        canvas.text(8, 12, "WEATHER", BLUE, 2)
        canvas.line(8, 42, 232, 42, DARK)

    def _draw_weather(self, canvas, snapshot):
        """绘制天气主体内容。"""
        weather = self._read_weather(snapshot)
        status = weather.get("status")
        if status != "ok":
            canvas.text(8, 72, "Waiting weather", YELLOW)
            canvas.text(8, 96, "Check plugin data", GRAY)
            return

        city = self._short_text(weather.get("city"), 18)
        text = self._short_text(weather.get("text"), 18)
        temperature = self._number(weather.get("temperature_c"))
        humidity = self._number(weather.get("humidity"))
        wind = self._number(weather.get("wind_kmh"))
        updated_at = self._short_text(weather.get("updated_at"), 8)

        canvas.text(8, 60, city, WHITE)
        canvas.text(8, 92, "{:4.1f} C".format(temperature), GREEN, 3)
        canvas.text(8, 136, text, WHITE, 2)
        canvas.fill_rect(8, 176, 224, 2, DARK)
        canvas.text(8, 194, "HUM {:3.0f}%".format(humidity), GRAY)
        canvas.text(120, 194, "WIND {:4.1f}".format(wind), GRAY)
        canvas.text(8, 218, "UPDATED " + updated_at, BLUE)

    @staticmethod
    def _draw_footer(canvas, snapshot):
        """绘制系统当前时间，方便判断屏幕仍在刷新。"""
        timestamp = str((snapshot or {}).get("timestamp", ""))
        clock = timestamp[11:19] if len(timestamp) >= 19 else "--:--:--"
        canvas.line(8, 280, 232, 280, DARK)
        canvas.text(8, 292, "LOCAL " + clock, GRAY)

    def draw_visible(self, canvas, snapshot):
        """绘制当前条带中可见的完整天气屏幕。"""
        snapshot = snapshot or {}
        canvas.clear(BLACK)
        self._draw_header(canvas)
        self._draw_weather(canvas, snapshot)
        self._draw_footer(canvas, snapshot)

    def draw_dirty(self, canvas, key, snapshot):
        """根据区域键重绘天气主体或页脚时钟。"""
        snapshot = snapshot or {}
        canvas.clear(BLACK)
        if key == "weather":
            self._draw_weather(canvas, snapshot)
        elif key == "footer":
            self._draw_footer(canvas, snapshot)


def create_weather_panel_style():
    """创建天气面板样式实例。"""
    return WeatherPanelStyle()


register_style(WeatherPanelStyle.name, create_weather_panel_style)
```

这个样式有两个动态区域：

| 区域 | 坐标 | 什么时候刷新 |
| --- | --- | --- |
| `weather` | `8, 52, 224, 180` | `snapshot.ext.weather` 改变时 |
| `footer` | `8, 282, 224, 28` | `snapshot.timestamp` 改变时 |

天气接口通常 10 分钟采集一次，页脚时间可能每秒变化。拆成两个区域后，天气主体不会被每秒重画，屏幕更稳定。

## 第九步：让插件失败时也返回可显示状态

如果你在旧版本示例中直接使用 `requests.get(...)`，接口失败时会抛出异常，Monitor 会记录错误，但 `snapshot.ext.weather` 可能保持旧值或暂时缺失。建议使用下面这种写法，让屏幕明确显示失败状态：

```python
def collect():
    """请求天气接口，失败时返回可显示的错误状态。"""
    params = {
        "latitude": LATITUDE,
        "longitude": LONGITUDE,
        "timezone": TIMEZONE,
        "current": "temperature_2m,relative_humidity_2m,weather_code,wind_speed_10m",
    }
    try:
        session = requests.Session()
        session.trust_env = USE_SYSTEM_PROXY
        response = session.get(API_URL, params=params, timeout=(3, 6))
        response.raise_for_status()
        payload = response.json()
        current = payload.get("current") or {}
        weather_code = current.get("weather_code")
        return {
            "city": CITY_NAME,
            "temperature_c": current.get("temperature_2m"),
            "humidity": current.get("relative_humidity_2m"),
            "wind_kmh": current.get("wind_speed_10m"),
            "code": weather_code,
            "text": _weather_text(weather_code),
            "updated_at": _hour_minute(current.get("time")),
            "status": "ok",
        }
    except requests.RequestException as exc:
        return {
            "city": CITY_NAME,
            "text": "Network error",
            "updated_at": dt.datetime.now().strftime("%H:%M"),
            "status": "error",
            "message": str(exc)[:80],
        }
```

屏幕样式中的 `status != "ok"` 分支会显示等待或错误提示，不会因为字段缺失崩溃。

`USE_SYSTEM_PROXY = False` 表示插件默认不继承系统代理和环境变量代理，适合代理配置不可用或代理握手超时的情况。如果你的网络必须通过代理访问外部接口，可以把它改成 `True` 后重新测试。

## 第十步：常见问题

| 现象 | 可能原因 | 处理方法 |
| --- | --- | --- |
| 自定义数据测试失败 | 网络不可用、代理限制、接口被拦截 | 在浏览器中打开接口地址，或查看“自定义数据”窗口底部日志 |
| 日志出现 `_prepare_proxy` 或代理握手超时 | 系统代理被 requests 自动读取但不可用 | 保持 `USE_SYSTEM_PROXY = False`，重新加载插件并测试 |
| `No module named requests` | 依赖没有安装成功 | 选中插件并重新单击“安装依赖” |
| 屏幕一直显示 `Waiting weather` | 插件还没有进入调度器或 `key` 不匹配 | 确认 `plugin.json` 的 `key` 是 `weather`，然后重启 Monitor |
| 温度显示为 `0.0 C` | 接口字段缺失或字段名写错 | 在插件测试结果中检查 `temperature_c` 是否存在 |
| 城市中文显示为 `?` | 点阵字体不包含对应字形 | 使用英文城市名，或换成已确认可显示的字符 |
| 上传样式被拒绝 | 文件名、BOM、大小或必需方法不符合规则 | 对照第 6 课的上传检查清单 |
| 天气主体频繁闪烁 | 把天气和时钟放进了同一个脏区域 | 按本课示例拆分 `weather` 和 `footer` 区域 |

## 扩展练习

可以在这个基础上继续改：

- 增加 `apparent_temperature`，显示体感温度。
- 增加 `precipitation_probability`，显示降雨概率。
- 把 `WEATHER_TEXT` 改成更短的屏幕文案，例如 `Cloud`、`Rain`、`Snow`。
- 根据温度改变颜色：低温用 `BLUE`，舒适用 `GREEN`，高温用 `YELLOW`。
- 创建第二个插件 `air_quality`，把空气质量放进 `snapshot.ext.air_quality`。

进阶时仍然遵守一个原则：插件负责把复杂接口整理成稳定小字段，屏幕样式只负责安全读取和绘制。

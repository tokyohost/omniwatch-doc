# Dev 模式使用说明

[返回总目录](../README.md) · [查看日志](08-查看日志.md) · [读取监控数据](02-读取监控数据.md)

Dev 模式用于开发和诊断电脑端采集数据。它最有价值的能力是：**即使手边没有 OmniWatch 硬件，也能持续采集电脑指标，并把本来要发送给设备的完整 JSON 打印到日志。**

## 开启后会发生什么

程序启动时会在日志中显示：

```text
开发模式=开启
```

随后有两种情况：

### 已连接 OmniWatch

程序正常采集和发送数据，同时额外输出一行完整数据：

```text
[DEV][Monitor -> Pico][JSON] JSON:{...}
```

这适合确认你的 Style 想读取的字段是否真实存在、字段名是否正确、值是什么类型。

### 没有找到 OmniWatch

首次串口搜索失败后，程序不再继续寻找设备，而是按照采集间隔持续打印 `[DEV][Monitor -> Pico][JSON]`。

这意味着：

- 可以无硬件开发和检查数据采集。
- 程序不会在 Dev 模式循环中自动发现后来插入的设备。
- 插入设备后，需要关闭 Dev 模式或重启后台监控，才能重新执行设备搜索。

## Windows 托盘中开关 Dev 模式

1. 右键单击 OmniWatch 托盘图标。
2. 单击“Dev 模式”。
3. 菜单前出现勾选表示已开启；再次单击表示关闭。
4. 切换后程序会保存设置，并自动重启后台监控进程。
5. 打开“日志”，搜索最新的“监控服务启动”行，确认其中是 `开发模式=开启` 还是 `开发模式=关闭`。

判断真实状态时以启动日志为准，不要只看旧日志中的 `[DEV]`，也不要只凭菜单记忆。

Windows 托盘配置保存在：

```text
%LOCALAPPDATA%\PicoMonitor\settings.json
```

一般不需要手工编辑这个文件，使用托盘开关更安全。

## 从源码显式开启

进入 `pico-project\monitor` 目录后运行：

```powershell
python pico_monitor.py --dev
```

只采集并输出一次后退出：

```powershell
python pico_monitor.py --dev --once
```

调整数据采集间隔为 1 秒：

```powershell
python pico_monitor.py --dev --interval 1.0
```

也可以使用环境变量。PowerShell 当前窗口示例：

```powershell
$env:PICO_MONITOR_DEV = "1"
python pico_monitor.py
```

环境变量只影响从该终端启动的进程。调试结束后可执行：

```powershell
Remove-Item Env:PICO_MONITOR_DEV
```

Linux 当前终端示例：

```bash
PICO_MONITOR_DEV=1 python pico_monitor.py
```

## 用 Dev 日志确认 Style 字段

假设样式准备读取：

```python
percent = snapshot.get("cpu", {}).get("percent", 0)
```

操作步骤：

1. 开启 Dev 模式。
2. 打开日志。
3. 搜索 `[DEV][Monitor -> Pico][JSON]`。
4. 在 JSON 中搜索 `"cpu"`。
5. 确认 `cpu` 内是否有 `percent`，以及它是数字、空值还是缺失。
6. 再决定 Style 中使用什么默认值和格式。

同理，想使用网络或磁盘数据时，先在真实 JSON 中找到对应层级，不要凭感觉猜字段名。

## Dev 模式能做什么、不能做什么

| 能做 | 不能做 |
| --- | --- |
| 无设备查看电脑采集数据 | 无设备预览 LCD 最终像素效果 |
| 核对 snapshot 字段和类型 | 无设备验证串口通信是否正常 |
| 观察真实发送 JSON | 自动发现后来才插入的设备 |
| 辅助排查数据缺失 | 代替样式上传器的结构校验 |

Dev 模式主要调试“数据从电脑端出来了吗”；样式文件是否合法仍要由上传校验器检查，实际布局和颜色最终还要在设备屏幕上确认。

## 什么时候应该关闭

- 正常日常使用，不需要查看完整 JSON 时。
- 正在排查设备自动重连或热插拔时。
- 希望程序在没有设备时持续重新搜索串口时。
- 完整 JSON 让日志增长太快、干扰普通排错时。

关闭后查看最新启动日志，确认出现 `开发模式=关闭`。如果没有变化，退出托盘程序后重新启动，再检查一次。

## 常见误区

**“开启 Dev 模式会把开发版固件刷进设备。”**

不会。它只改变电脑端监控程序的采集、日志和无设备运行行为。

**“日志里有 JSON，就说明屏幕一定能正常显示。”**

不一定。JSON 只证明电脑端生成了数据；还需要设备连接、协议确认和 Style 正常渲染。

**“先启动 Dev 模式，稍后插设备也会自动连上。”**

首次搜索失败并进入无设备开发循环后不会继续探测。插入设备后请重启后台监控，或先关闭 Dev 模式。

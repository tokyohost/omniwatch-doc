## snapshot 是什么

可以把 `snapshot` 想成下面这只“数据柜”：

```python
{
    "host": "MY-PC",
    "timestamp": "2026-07-07 16:30:45",
    "cpu": {
        "percent": 37.5,
        "history": [20, 25, 31, 37.5]
    },
    "network": {
        "online": True
    }
}
```

实际数据可能暂时缺失，所以不要写 `snapshot["cpu"]["percent"]`。这会在某个键不存在时直接报错。推荐逐层使用 `.get()`：

```python
snapshot = snapshot or {}
cpu = snapshot.get("cpu", {})
percent = cpu.get("percent", 0)
```

## 不要相信数据一定是数字

设备刚连接、采集器异常或字段为空时，CPU 值可能是 `None` 或文本。完整示例用一个辅助方法兜底：

```python
@staticmethod
def _number(value, default=0):
    """把未知输入安全转换为浮点数。"""
    try:
        return float(value)
    except (TypeError, ValueError):
        return float(default)
```

然后限制到合理范围：

```python
percent = max(0, min(100, self._number(cpu.get("percent"))))
```

## 格式化适合屏幕的文字

小屏幕要主动控制长度：

```python
host = str(snapshot.get("host", "WAITING"))[:18]
cpu_text = "CPU {:3.0f}%".format(percent)
timestamp = str(snapshot.get("timestamp", ""))
clock = timestamp[11:19] if len(timestamp) >= 19 else "--:--:--"
```

- `[:18]` 最多保留 18 个字符。
- `{:3.0f}` 表示总宽度至少 3 位、不显示小数。
- 时间格式不完整时显示占位符，不让界面崩溃。

## 中英文显示提醒

内置点阵字体只包含它支持的字形；找不到的字符会显示成 `?`。界面标题可用英文缩写，代码注释和 `zh_name` 使用中文。可选字体有：

- `native`：默认字体，最省资源。
- `screen_2inch`：适配 2 英寸屏幕的字体。
- `screen_2inch_compact`：更紧凑，适合信息较多的布局。

# 第 10 课：自定义数据采集

[上一课](09-Dev模式.md) · [返回总目录](../README.md)

自定义数据脚本运行在电脑端 Monitor 中，适合把你自己的传感器、HTTP 接口、脚本计算结果或本地状态加入发送给 Pico 的 `snapshot`。每个 `.py` 文件会在 Monitor 启动时封装成一个独立采集任务，可以和 CPU、磁盘、网络任务一样单独配置采集间隔。

## 数据会出现在什么位置

脚本返回的数据会合并到 `snapshot["ext"]` 下：

```python
{
    "ext": {
        "demo_sensor": {
            "temperature_c": 27.3,
            "status": "ok"
        }
    }
}
```

多个自定义数据脚本会分别更新自己的字段，不会互相覆盖。

## 最小脚本

请复制 [custom_data_demo.py](code/custom_data_demo.py)。核心结构如下：

```python
CUSTOM_DATA_KEY = "demo_sensor"
CUSTOM_DATA_NAME = "demo_sensor"
CUSTOM_DATA_ZH_NAME = "演示传感器"
CUSTOM_DATA_INTERVAL = 5


def collect():
    """返回一个可以 JSON 序列化的对象。"""
    return {
        "temperature_c": 27.3,
        "status": "ok",
    }
```

三个名称不要混用：

| 字段 | 作用 | 示例 |
| --- | --- | --- |
| `CUSTOM_DATA_KEY` | 写入 `snapshot.ext` 下的数据字段名 | `demo_sensor` |
| `CUSTOM_DATA_NAME` | 采集任务英文标识，用于配置间隔 | `demo_sensor` |
| `CUSTOM_DATA_ZH_NAME` | Windows 配置页和日志展示名 | `演示传感器` |

最终配置采集间隔时，完整任务 key 是：

```yaml
custom_data.demo_sensor: 5
```

其中 `custom_data.` 是固定前缀，后面的 `demo_sensor` 来自 `CUSTOM_DATA_NAME`。

## 字段规范

`CUSTOM_DATA_KEY` 和 `CUSTOM_DATA_NAME` 必须满足：

- 只能包含英文字母、数字和下划线。
- 不能以数字开头。
- 长度不要超过 64 个字符。
- 在所有自定义数据脚本中唯一。

`CUSTOM_DATA_ZH_NAME` 可以写中文，建议短一些，方便在配置页和日志中阅读。

旧脚本仍可运行：如果没有 `CUSTOM_DATA_NAME`，Monitor 会默认使用 `CUSTOM_DATA_KEY`；如果没有 `CUSTOM_DATA_ZH_NAME`，会默认使用英文名。但新脚本建议三个字段都写清楚。

## Windows 加载脚本

1. 右键托盘图标。
2. 打开“自定义数据”。
3. 单击“加载 py 文件”。
4. 选择你的自定义数据脚本。
5. 列表中确认 “JSON Key”“任务 Key”“中文名称”“间隔(秒)”。
6. 单击“测试”，确认底部能看到 JSON 输出。
7. 关闭窗口后，打开“配置”。
8. 在“系统采集任务”中找到你的中文名称，例如 `演示传感器（custom_data.demo_sensor，默认 5 秒）`。
9. 修改采集间隔并保存配置，Monitor 会重启后生效。

Windows 配置页保存的是英文任务 key，中文名称只用于显示。

## Linux 配置采集间隔

Linux 服务配置位于：

```text
/etc/pico-monitor.conf
```

在 `collection_tasks.intervals` 中加入或修改自定义数据任务：

```yaml
collection_tasks:
  intervals:
    cpu_memory: 1
    network: 1
    custom_data.demo_sensor: 5
```

保存后重启服务：

```bash
sudo systemctl restart pico-monitor
```

如果脚本中 `CUSTOM_DATA_INTERVAL = 5`，配置里又写了 `custom_data.demo_sensor: 2`，最终以配置文件中的 `2` 秒为准。

## 在样式中读取自定义数据

自定义屏幕样式中按普通 `snapshot` 字段读取即可：

```python
ext = snapshot.get("ext", {})
sensor = ext.get("demo_sensor", {})
temperature = sensor.get("temperature_c")
status = sensor.get("status", "unknown")
```

请永远用 `.get()` 读取。脚本刚加载、首次采集还没完成、脚本报错或字段暂时缺失时，`ext.demo_sensor` 可能不存在。

## collect 返回值要求

`collect()` 必须返回可以被 JSON 序列化的数据，例如：

- `dict`
- `list`
- `str`
- `int` / `float`
- `bool`
- `None`

不要返回文件句柄、数据库连接、对象实例、字节流等无法转成 JSON 的值。需要传二进制数据时，先转成字符串，例如 Base64 或十六进制文本。

## 常见错误

| 提示或现象 | 原因 | 修复方法 |
| --- | --- | --- |
| `必须定义非空字符串 CUSTOM_DATA_KEY` | 没写数据字段名 | 补上 `CUSTOM_DATA_KEY = "xxx"` |
| `CUSTOM_DATA_NAME 重复` | 两个脚本使用同一个任务名 | 给其中一个脚本换唯一英文名 |
| `CUSTOM_DATA_KEY 重复` | 两个脚本写入同一个 `ext` 字段 | 修改其中一个 `CUSTOM_DATA_KEY` |
| `collect 返回值不是合法 JSON 数据` | 返回了不能 JSON 化的对象 | 改成字典、列表、数字、字符串等基础类型 |
| 配置里找不到自定义任务 | 脚本未加载或 Monitor 未重启 | 在自定义数据窗口测试脚本，再重启 Monitor |
| 屏幕读不到数据 | 首次采集未完成或字段名写错 | 对照 `CUSTOM_DATA_KEY`，并用 `.get()` 兜底 |

## 建议做法

一个脚本只负责一种数据来源。例如天气、NAS 状态、家庭传感器分成三个脚本，而不是塞进一个大脚本。这样每个任务都能单独配置采集间隔，也更容易定位错误。

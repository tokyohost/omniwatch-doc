## 上传前一分钟检查

假设样式中写的是：

```python
name = "my_chart"
zh_name = "我的图表"
type = "custom"
```

那么文件必须叫 `style_my_chart.py`。再逐项确认：

- [ ] 编码是 UTF-8（无 BOM）。
- [ ] 文件大小不超过 16384 字节。
- [ ] 只有一个 `type = "custom"` 的样式类。
- [ ] 类中有三个必需方法。
- [ ] 文件末尾恰好调用一次 `register_style`。
- [ ] `name` 没有中文、大写字母、短横线或空格。
- [ ] 所有类和方法都有清楚的中文文档注释。

## 在电脑端上传

1. 连接 OmniWatch，并确认电脑端程序正在运行。
2. 在托盘菜单中打开“自定义屏幕”。
3. 等待窗口显示设备的自定义样式清单和 Flash 剩余空间。
4. 单击“上传文件”。
5. 选择你的 `style_*.py` 文件。
6. 查看“上传日志”；成功后清单会自动刷新。
7. 如果设备已有同名样式，确认是否覆盖。
8. 回到样式选择界面，选择刚上传的中文名称。

上传前电脑端会做静态校验，所以许多错误不会真的写入设备。

## 常见错误对照表

| 提示或现象 | 原因 | 修复方法 |
| --- | --- | --- |
| 文件不能包含 UTF-8 BOM | 编辑器保存了 BOM | 改为 UTF-8 无 BOM 后重新保存 |
| 文件名必须为 `style_xxx.py` | 文件名与 `name` 不一致 | 严格按 `style_` + `name` 命名 |
| 样式名仅允许…… | `name` 有大写、中文、空格或 `-` | 只用 `a-z`、`0-9`、`_` |
| 缺少必要方法 | 三个核心方法不全或缩进到了类外 | 对照完整示例补齐并检查缩进 |
| `register_style` 必须且只能注册…… | 缺失、重复或注册了错误类 | 文件末尾只保留一次正确注册 |
| 脏矩形超过画布容量 | 某动态区域面积超过约 9600 | 缩小区域或拆成上下两块 |
| 屏幕有残影 | 脏区域过小或未清背景 | 扩大区域，`draw_dirty` 先 `clear` |
| 文字显示 `?` | 当前点阵字体没有这个字形 | 换支持的字符或其他内置字体 |
| 内容位置错乱 | 把条带坐标当成屏幕坐标 | 始终使用完整屏幕绝对坐标 |
| 上传后回退默认样式 | 自定义样式运行时异常 | 查看日志，从数据缺失和区域容量查起 |

## 二分法排错

复杂样式出错时，不要盯着几百行代码硬猜：

1. 先备份文件。
2. 暂时只保留背景和一个标题。
3. 确认可以上传并显示。
4. 每次恢复一个区域。
5. 某一步重新出错，问题就在刚恢复的区域。

这是最笨也最可靠的方法。屏幕样式的 bug 很会躲，但它不会魔法。

仍然无法解决时，导出日志并保留问题发生前后几十行。提交问题时一并说明操作系统、样式文件名和复现步骤，比只说“不能用”有效得多。

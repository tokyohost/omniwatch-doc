# 第 4 课：局部刷新，让屏幕更流畅

[上一课](03-布局颜色与进度条.md) · [返回总目录](../README.md) · [下一课](05-图表与性能.md)

每秒把整屏全部重画一次既浪费时间，也可能产生闪烁。OmniWatch 会比较前后两份 `snapshot`，只重画发生变化的小区域。

## 第一步：声明动态区域

```python
@staticmethod
def create_dirty_regions():
    """声明可独立刷新的动态区域。"""
    return [
        ("header", 8, 8, 224, 28),
        ("cpu", 8, 48, 224, 72),
        ("network", 8, 136, 224, 32),
        ("footer", 8, 292, 224, 18),
    ]
```

每一项依次是 `(唯一键, x, y, 宽, 高)`。区域必须覆盖旧内容可能占据的全部范围，因为重绘前会清空整个区域。

> 单个脏区域不能超过条带画布容量。默认容量约等于 `240 × 40` 像素，即面积不能超过 9600 像素。CPU 卡片 `224 × 72` 太大，完整示例实际把动态内容限制在较小区域；大型图表应拆成上下两块。

## 第二步：选择真正变化的区域

`select_dirty_regions` 不是校验器强制的方法，但实现它才能精准刷新：

```python
@classmethod
def select_dirty_regions(cls, previous, current):
    """比较前后数据并返回需要刷新的区域。"""
    previous = previous or {}
    current = current or {}
    regions = {item[0]: item for item in cls.create_dirty_regions()}
    selected = []
    if previous.get("cpu") != current.get("cpu"):
        selected.append(regions["cpu"])
    if previous.get("network") != current.get("network"):
        selected.append(regions["network"])
    return selected
```

更细致的做法是只比较 `cpu.percent`，避免历史数据变化时误刷 CPU 数值区。完整代码已经这样处理。

## 第三步：只画指定区域

```python
def draw_dirty(self, canvas, key, snapshot):
    """根据区域键重绘一块动态内容。"""
    snapshot = snapshot or {}
    canvas.clear(BLACK)
    if key == "cpu":
        self._draw_cpu(canvas, snapshot)
    elif key == "network":
        self._draw_network(canvas, snapshot)
```

渲染器已把 Canvas 视口裁到目标区域。`canvas.clear(BLACK)` 只会清理这个区域，不会把整屏擦黑。

## 最常见的残影原因

- 脏区域只覆盖新文字，没有覆盖更长的旧文字。
- `draw_dirty` 没先清背景。
- `key` 拼写与 `create_dirty_regions` 中不一致。
- 数据变了，但 `select_dirty_regions` 忘了返回对应区域。

排查时可暂时把脏区域背景改成显眼颜色，观察它是否完整覆盖动态内容，确认后再改回 `BLACK`。

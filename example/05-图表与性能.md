## 用一个调用绘制整张图

```python
canvas.draw_line_chart({
    "x": 8,
    "y": 112,
    "width": 224,
    "height": 76,
    "maximum": 100,
    "color": BLUE,
    "filled": True,
    "regions": ((60, GREEN), (85, YELLOW), (101, BLUE)),
    "grid_step_x": 16,
    "grid_step_y": 12,
    "grid_color": GRAY,
}, values or ())
```

参数解释：

| 参数 | 含义 |
| --- | --- |
| `x, y, width, height` | 图表所在矩形 |
| `maximum` | 纵轴最大值；CPU 使用率固定为 100 |
| `filled` | `True` 画面积图，`False` 画折线 |
| `regions` | 按数值区间选择颜色，按顺序匹配第一个上限 |
| `grid_step_x/y` | 网格点间距；设为 0 可关闭 |

`history` 可能不存在，所以使用：

```python
values = snapshot.get("cpu", {}).get("history", ())
```

## 大图表为什么要拆脏区域

图表是 `224 × 76 = 17024` 像素，超过约 9600 像素的画布容量。声明局部刷新时拆成两块：

```python
("history_top", 8, 112, 224, 38)
("history_bottom", 8, 150, 224, 38)
```

两块的 `draw_dirty` 都调用同一个 `_draw_history`。Canvas 会自动裁剪，只绘制当前半块。

## 性能习惯

- 优先使用 `draw_line_chart`、`draw_commands` 等批量接口。
- 不要在每一帧创建无限增长的列表或字典缓存。
- 不要在样式中导入 `fn_canvas`，也不要判断当前是不是 C 后端。
- 重复显示的文本可在 `prepare_frame(snapshot)` 中预先格式化。
- 一个复杂区域先判断是否与当前条带相交，再绘制，可减少重复计算。
- 文件必须小于 16 KB；样式是界面，不宜承担数据采集或业务处理。

## `prepare_frame` 什么时候用

当同一帧的多个条带都会使用相同文本时，可提前准备：

```python
def prepare_frame(self, snapshot):
    """在绘制前集中准备本帧要显示的文字。"""
    snapshot = snapshot or {}
    percent = self._number(snapshot.get("cpu", {}).get("percent"))
    self._cpu_text = "CPU {:.0f}%".format(percent)
```

这不是必需方法。简单样式优先保持清楚，遇到真实性能问题再加缓存。
